# Security Policy

Better Life Naver는 계정 세션, API 키, Electron IPC, 브라우저 자동화, 외부 콘텐츠 수집 및 자동 업데이트를 다룹니다. 보안 문제를 발견했다면 악용 가능성을 낮추기 위해 아래 절차를 따라 주세요.

## 지원 범위

보안 수정은 다음 범위를 우선 지원합니다.

| 버전 | 지원 상태 |
| --- | --- |
| 최신 GitHub 릴리스 | 지원 |
| `main` 브랜치 | 지원 |
| 이전 릴리스 | 중요한 취약점에 한해 최선의 노력으로 지원 |

## 비공개 제보 방법

1. 이 저장소의 **Security** 탭에서 **Report a vulnerability** 또는 새 비공개 보안 권고 초안을 사용해 주세요.
2. 비공개 제보 버튼이 보이지 않으면 취약점 세부 내용 없이 `Security contact request`라는 제목의 Issue를 만들어 비공개 연락 방법을 요청해 주세요.
3. 공개 Issue, Discussion, Pull Request에는 실제 비밀정보, 재현 가능한 공격 코드, 피해 사용자 정보 또는 아직 수정되지 않은 취약점 세부 내용을 게시하지 마세요.

제보에는 가능한 범위에서 다음 정보를 포함해 주세요.

- 영향을 받는 버전과 운영체제
- 문제가 발생하는 구성요소와 공격 전제조건
- 비밀정보를 제거한 최소 재현 절차
- 예상 영향과 제안하는 완화 방법
- 공개 가능한 시점에 대한 의견

## 우선 검토 영역

- API 키, 계정 정보 및 세션의 저장·마스킹·암호화
- Electron 메인/렌더러 프로세스 사이의 IPC 경계
- 외부 URL, RSS, HTML 및 이미지 입력 검증
- 프롬프트 인젝션, XSS, SSRF 및 경로 조작
- 브라우저 자동화 세션과 다중 계정 격리
- 자동 업데이트, 설치 파일 및 릴리스 공급망
- 로그, 오류 메시지 및 진단 산출물의 민감정보 노출

## 처리 원칙

- 제보 접수 후 재현 가능성과 영향을 먼저 확인합니다.
- 수정이 준비될 때까지 필요한 정보만 최소 인원과 공유합니다.
- 가능한 경우 회귀 테스트와 함께 수정하고 최신 릴리스에 반영합니다.
- 공개 시점에는 제보자와 협의하고, 원한다면 기여를 명시합니다.

## 안전한 연구

본인이 소유하거나 명시적으로 허가받은 계정과 데이터만 사용해 주세요. 서비스 방해, 대량 자동화, 실제 사용자 데이터 접근, 자격 증명 탈취 또는 제3자 시스템 공격은 허용되지 않습니다.
