# Better Life Naver v2.11.35 업데이트 안내

이번 업데이트는 눈에 보이는 새 버튼보다, 실제 사용 중 갑자기 흐름이 끊기거나 글 품질이 흔들리는 문제를 줄이는 안정화 업데이트입니다.

## 핵심 개선

### 1. 글 생성 파이프라인 안정화
- 글 생성 엔진 내부 구조를 더 작은 모듈로 분리해 한 기능 수정이 다른 모드에 영향을 주는 위험을 줄였습니다.
- SEO, 홈판, 쇼핑커넥트, 네이버 메이트 등 여러 모드에서 공통으로 쓰는 정리 로직을 별도 검증 모듈로 분리했습니다.
- 전체 테스트 3190개를 통과한 상태로 릴리스했습니다.

### 2. 이상 문구 자동 정리 강화
- AI가 가끔 본문에 섞어 넣는 HTML 태그, 출처를 꾸며낸 듯한 표현, 자가검수 문구를 발행 전 자동으로 정리합니다.
- 예를 들어 `<div>`, `<h4>`, `기사에 따르면`, `자가검수`처럼 블로그 본문에 들어가면 어색한 표현을 더 안정적으로 제거합니다.
- 네이버 에디터에 그대로 붙었을 때 사용자 손이 다시 가지 않도록 정리 단계를 강화했습니다.

### 3. 홈판 모드 검증 안정화
- 홈판 모드의 제목 길이, 감정 트리거, 소제목 수, 도입부 길이, 결론 표현, 키워드 밀도 검증을 별도 모듈로 분리했습니다.
- 소제목이 부족한 경우 주제와 관련된 보조 소제목을 자동 보강하는 흐름을 유지하면서 테스트로 고정했습니다.
- 홈판이 아닌 모드에서도 공통 sanitizer가 먼저 실행되도록 보장했습니다.

### 4. 대형 파일 리스크 감소
- 가장 큰 파일인 `contentGenerator.ts`를 약 418KB에서 약 395KB까지 줄였습니다.
- 글 생성, 검증, 정리 로직이 한 파일에 과도하게 몰려 있던 구조를 단계적으로 분리했습니다.
- 앞으로 수정할 때 특정 모드만 고치려다가 다른 모드가 같이 깨지는 회귀 가능성을 낮췄습니다.

### 5. 릴리스 검증
- `npm run build` 통과
- 전체 테스트 `287 files / 3190 tests` 통과
- Windows 설치 파일과 `latest.yml` GitHub 업로드 후 fresh-fetch 검증 통과
- 자동 업데이트 Latest 포인터가 `v2.11.35`를 가리키는 것까지 확인

## 사용자 체감

- 글 생성 결과에 이상한 태그나 내부 점검 문구가 섞일 가능성이 줄었습니다.
- 홈판/SEO/쇼핑커넥트 모드에서 글 구조 검증이 더 안정적으로 동작합니다.
- 수정 하나 때문에 다른 기능이 같이 흔들리는 위험을 줄이는 방향으로 내부 구조를 정리했습니다.
- 자동 업데이트 자산까지 검증된 상태로 배포했습니다.
