## v2.11.40 안정화 업데이트

이번 버전은 발행 직전 단계에서 창이 다시 열리거나, 세션 복구 후 진행률이 되돌아가며 실패하는 상황을 줄이기 위한 안정화 업데이트입니다.

### 좋아진 점

- 발행 모달이 정상적으로 열리지 않으면 더 이상 억지로 다음 단계로 넘어가지 않고 정확한 원인으로 중단합니다.
- 글쓰기 화면, 제목 입력칸, 본문 에디터, 발행 모달 상태를 더 촘촘하게 확인합니다.
- 이미지가 일부만 들어간 상태로 발행되는 문제를 막았습니다.
- 계획한 이미지 수와 실제 에디터에 들어간 이미지 수가 다르면 발행 전 검증에서 잡아냅니다.
- 이미지 삽입 중 일부 이미지가 실패하면 `IMAGE_INSERTION_FAILED`로 명확하게 분류합니다.
- 세션 복구나 브라우저 상태 이상이 생겼을 때 원인을 더 빨리 파악할 수 있도록 로그와 실패 분류를 강화했습니다.

### 사용자에게 보이는 변화

- 95% 부근에서 88%로 되돌아가며 다시 작성하다가 실패하는 흐름이 줄어듭니다.
- 이미지가 빠진 글이 그대로 발행되는 위험이 줄어듭니다.
- 실패하더라도 "왜 실패했는지"를 더 명확하게 확인할 수 있습니다.
- 반자동, 풀오토, 연속발행, 다중계정 발행에서 발행 직전 검증이 더 견고해졌습니다.

### 참고

네이버 로그인 세션 만료, 네이버 글쓰기 UI 변경, 네트워크 끊김, PC 절전/백신/크롬 강제 종료 같은 외부 변수는 완전히 없앨 수 없습니다. 대신 이번 버전부터는 그런 상황에서도 무리하게 계속 진행하지 않고, 더 정확한 원인 로그를 남기도록 개선했습니다.
