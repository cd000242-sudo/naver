## v2.11.100 안정화 릴리스

### 발행 안정성
- 반자동 편집 본문이 렌더러에서 메인 발행 엔진까지 원래 순서대로 전달되는 실제 UI 회귀 테스트를 추가했습니다.
- 최근 글이 없거나 20건 미만인 신규 사용자에게 내부 `CONTENT_POLICY_BLOCKED` 오류를 표시하던 문제를 수정했습니다. 글 생성과 품질 검사는 계속 진행하고, 발행 직전 검수 확인 후 같은 원고와 이미지로 이어집니다.
- 최근 글 검수 승인은 유사도 자료 부족에만 적용되며 제목·본문 불일치, 허위·금지 표현, 발행 중지 상태 등 실제 안전 차단은 우회하지 않습니다.
- 비어 있는 메인 저장소가 유효한 렌더러·기존 발행 이력을 가리던 폴백 문제를 수정하고, 앱 예약 글은 검수 승인 상태를 글별로 보존합니다.
- 즉시 발행 성공은 실제 네이버 게시물 URL이 확인된 경우에만 인정하며, 경로형 및 `blogId`/`logNo` 쿼리형 URL을 동일하게 처리합니다.
- 앱 예약 발행을 실제 로컬 예약 큐에 저장하고 예약 시각 기준 발행 간격 정책을 적용합니다.
- 미래의 먼 예약이 현재 발행 간격을 잘못 차단하지 않도록 이전·다음 예약 중 인접한 시각만 비교합니다.
- 앱 예약은 저장 당시 선택한 네이버 계정에 묶이며, 실행 시 다른 계정으로 바뀌어 있으면 오발행 대신 안전하게 중단합니다.
- 앱 예약과 스마트 스케줄러 실행도 라이선스와 무료 발행량을 검사하고 선차감하며, 실패 시에만 환불합니다. 결과가 불확실하면 중복 사용을 막기 위해 사용량을 유지합니다.
- 반복 예약은 이전 회차의 최근 글 검수 승인을 물려받지 않고 회차마다 새로 확인합니다.
- 예약 정책 원장 저장 실패 시 신규 예약을 제거하고, 기존 예약 수정 중이었다면 이전 예약을 복원합니다.

### 이미지 안정성
- 로컬 폴더 이미지 경로와 `file://` URL이 리더스 나노바나나 무제한 참조 이미지 업로드 단계까지 보존됩니다.
- 수집 이미지 중복 제거와 대표 이미지 선정을 강화했습니다.
- 요청한 이미지 일부만 생성된 경우 성공으로 넘기지 않고 전체 배치를 명확히 실패 처리합니다.

### 릴리스 방어선
- 전체 테스트와 빌드 외에 IPC 계약, 실제 Electron UI, 런타임 self-test를 필수 릴리스 게이트로 추가했습니다.
- 모든 Electron 테스트와 설치본 스모크는 실제 사용자 설정을 건드리지 않는 임시 프로필에서 실행됩니다.
- GitHub 업로드 전에 Windows 패키지 앱을 직접 부팅하고 IPC 상태를 확인하며, 실패하면 릴리스를 중단합니다.
