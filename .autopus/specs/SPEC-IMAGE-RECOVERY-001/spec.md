## 1. 배경

ImageFX/Flow 이미지 생성에서 사용자별로 30+ 종류의 실패 분기가 코드에 명시돼 있다.
일부는 이미 자동 복구되지만, 약 절반은 **에러 메시지만 띄우고 사용자에게 전가**된다.
3일째 사용 불가 사례가 발생해, 앱이 단독으로 풀 수 있는 영역을 모두 자동화하고
못 푸는 영역은 **차단형 모달로 명확하게 안내**하는 것이 본 SPEC의 범위다.

본 SPEC은 메모리에 저장된 두 원칙을 엄수한다:
- **자동 폴백 금지** (feedback_no_fallback): 사용자 선택 모델 실패 시 다른 엔진/모델로 silent 전환 절대 금지.
  단 **같은 엔진 내에서 같은 모델로 자동 복구**(재로그인, 재시도, 백오프, 셀렉터 폴백)는 허용.
- **추정 효과 금지** (feedback_no_speculation): 본 SPEC에는 검증 안 된 효과 수치(% 등) 일체 기재 금지.

## 2. 목적

| 분류 | 목표 |
|------|------|
| **자동 복구** | 앱이 단독으로 해결 가능한 일시적 실패는 사용자 개입 없이 회복 |
| **명확 차단** | 외부 조건(IP·계정·2FA·UI 변경 등) 실패는 차단형 모달로 정확한 다음 행동 제시 |
| **silent 금지** | 다른 엔진으로 자동 폴백 절대 금지. 같은 엔진 내 복구만 허용 |
| **격리** | 한 헤딩 실패가 전체 배치를 끌어내리지 않게, 회복 불가능 에러만 즉시 중단 유지 |

## 3. 범위

### 3.1 In-Scope (자동화 대상)

| ID | 증상 | 자동 조치 | 엔진 |
|----|------|----------|------|
| **R1** | Google 세션 401 만료 | 토큰 캐시 폐기 + 1회 자동 재로그인 (현재 에러만 노출) | ImageFX/Flow |
| **R2** | 일시 네트워크 타임아웃 | 헤딩 단위 지수 백오프 3회 재시도 후 격리 | ImageFX/Flow |
| **R3** | AdsPower 양쪽 실패 케이스 | `adspower_enabled` 자동 OFF 토스트 + Playwright 단독 폴백 | ImageFX/Flow |
| **R4** | Flow 셀렉터 변경 (F1/F3/F5) | 다중 셀렉터 배열 + remoteUpdate 강제 갱신 시도 | Flow |
| **R5** | Google 로그인 진행 중 timeout | URL/DOM 변화 감지로 활성도 폴링 (정상 진행 중이면 5분→8분 자동 연장, 30분 상한) | ImageFX/Flow |
| **R6** | 이미지 다운로드 < 1024바이트 (D1/F6) | 5초 후 1회 재요청 (현재 즉시 reject) | ImageFX/Flow |
| **R7** | aHash 계산 실패 시 SHA256 단독 비교 (E4) | 이미 동작 — 진단 로그 강화만 | ImageFX/Flow |
| **R8** | 마라톤 콜드 스타트 정리 실패 | 이미 무시 — 경고 카운터 추가로 누적 추적 | Flow 마라톤 |

### 3.2 Out-of-Scope (앱이 단독 해결 불가 → 차단형 모달)

| ID | 증상 | 모달 안내 |
|----|------|----------|
| **B1** | HTTP 403 한국 IP 차단 (IMAGEFX_FORBIDDEN) | "외부 프록시 등록 후 재시도" + 가이드 링크 |
| **B2** | 안전 필터 차단 (B4) | "프롬프트에 다음 단어 포함됨: …. 수정하시겠습니까?" |
| **B3** | HTTP 429 시간당 한도 (B1) | "1시간 대기 / 다른 Google 계정 / 대안 엔진 수동 선택" 3택 모달 |
| **B4** | Playwright Chromium 4방법 모두 실패 (A1) | Chrome/Edge 다운로드 링크 + "관리자 권한으로 앱 실행" |
| **B5** | Google 로그인 30분 한도 초과 (R5 후) | "2FA 푸시 거부 가능성. 다시 로그인하시겠습니까?" |
| **B6** | Flow UI 변경으로 셀렉터 다중 폴백도 실패 (R4 후) | "Flow 페이지 변경 감지. 원격 패치 1시간 내 배포 예정 / 다른 엔진 수동 전환" |
| **B7** | 회복 불가능 에러 (QUOTA/FORBIDDEN/AUTH_EXPIRED) | 배치 즉시 중단 + 정확한 원인 모달 |

### 3.3 Non-Goals

- VPN 자동 활성화 (앱 권한 밖)
- 다른 엔진(나노바나나/덕트테이프/Gemini)으로 자동 전환 (silent 폴백 금지)
- 이미지 생성 모델 변경 (사용자 선택값 100% 존중)
- ImageFX 한국 IP 차단을 우회하는 어떤 종류의 자동화

## 4. 핵심 요구사항

### 4.1 자동 복구 정책

- **R-1**: 자동 복구는 **같은 엔진 같은 모델**에서만 허용. 엔진/모델 전환은 항상 차단형 모달.
- **R-2**: 자동 복구 시 사용자에게 토스트로 1회 알림 (silent 금지).
  - 형식: `🔄 [엔진] 일시 오류 자동 복구 (재시도 N/3): {원인}`
- **R-3**: 자동 복구 실패가 N회 누적되면 차단형 모달로 격상.
- **R-4**: 회복 불가능 에러(QUOTA/FORBIDDEN/AUTH_EXPIRED/UI변경)는 즉시 차단형 모달.

### 4.2 차단형 모달 정책

- **M-1**: 모든 차단형 모달에 **정확한 원인 코드** + **다음 행동 3택 이내** 표시.
- **M-2**: 모달은 사용자 명시 동의(버튼 클릭) 없이는 자동으로 닫히거나 다음 동작 진행 금지.
- **M-3**: "다른 엔진으로 자동 전환" 버튼 절대 만들지 않음. 사용자가 설정에서 직접 변경 유도.
- **M-4**: 모달 노출 시점에 진행 중인 배치 상태 저장 (재개 가능 체크포인트).

### 4.3 격리 정책

- **I-1**: 헤딩 단위 실패는 다음 헤딩으로 격리 (현재 정책 유지).
- **I-2**: 회복 불가능 에러 또는 3연속 실패는 즉시 배치 중단 (현재 정책 유지).
- **I-3**: 자동 복구 시도 횟수는 **헤딩 단위로 리셋**(이전 헤딩에서 R1~R6 발생해도 다음 헤딩은 깨끗하게 시작).

### 4.4 진단 로그 정책

- **D-1**: 자동 복구 성공 시 `[RECOVERY:OK]` 태그로 로그.
- **D-2**: 차단형 모달 노출 시 `[RECOVERY:HALT]` 태그 + 원인 코드 + 사용자 선택 결과 로그.
- **D-3**: 운영 대시보드(`src/monitor/operationsDashboard.ts`)에 복구 메트릭 8개 집계 추가:
  - R1~R8 각 발생 횟수, 평균 복구 소요 시간, 최종 차단 모달 비율.

## 5. 비요구사항 (명시적 거부)

- ❌ "다른 엔진으로 자동 전환" 옵션
- ❌ 사용자 모르게 silent 재시도 (모든 복구는 토스트 알림 필수)
- ❌ 한국 IP 차단을 우회하는 프록시 자동 임의 설정 (등록된 외부 프록시만 사용)
- ❌ Google 비밀번호/2FA 코드 자동 입력 (사용자만)
- ❌ 본 SPEC 외 영역(예: 콘텐츠 생성, 발행)에 영향 주는 변경

## 9. 위험

- **R-Risk-1**: 자동 재시도가 Google API 한도(429)를 더 빨리 소진할 수 있음 → R-3로 누적 가드.
- **R-Risk-2**: 다중 셀렉터(R4)로 Flow 페이지 진입은 늘어나지만 잘못된 요소 클릭 가능 → 통합 테스트 필수.
- **R-Risk-3**: 차단형 모달이 너무 자주 뜨면 사용자 피로도 증가 → 운영 대시보드(D-3)로 발생률 모니터링.
