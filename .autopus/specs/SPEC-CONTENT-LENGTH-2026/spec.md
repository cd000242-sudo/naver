# SPEC-CONTENT-LENGTH-2026: 본문 글자수 분야별 가이드 (체류시간 4분 도달)

## 1. 목적

사용자 심층 리서치 보고서 v2.0(2026-05-26 수령) Agent C "본문 최적화 전문가" 섹션의 **분야별 실측 글자수 범위**를 본문 생성 흐름에 반영하여, MDE Ranker가 평가하는 체류시간(평균 1분 → 4분 목표)을 끌어올린다.

본 SPEC은 작업 1+2+3(제목 프롬프트 R1~R3)과 별도로 분리한다. **본문 생성 흐름은 제목과 영향 영역이 다르며, contentGenerator.ts의 본문 합성 로직을 손대야 하므로 회귀 cascade 위험이 큼**. 단일 릴리즈에 묶지 않는다.

## 2. 배경

### 보고서 핵심 (Agent C, 2026-05-26)

**분야별 권장 글자수**:

| 분야 | 권장 글자 수 | 이유 |
|---|---|---|
| 정보형 (재테크·법·의학·정책) | 1,800~2,500자 | AI 브리핑 우회용 충실 정보 + 체류시간 확보 |
| 경험·후기형 (제품 리뷰·여행기) | 1,200~1,800자 | 몰입형 스토리텔링 길이, 4분 체류 가능 |
| 시즌·이슈형 (시즌 추천·이슈 분석) | 800~1,200자 | 빠른 결말 → Click2Click 다음 글 유도 |
| 펀팩트·일상형 (잡담·짧은 팁) | 600~900자 | 모바일 한 화면 분량, 빠른 소비 |

**근거**:
- 휘나리 분석(2026.02.23) — "체류시간 평균 1분 → 4분으로 뛰면서 상위 노출 유지력 완전히 달라짐"
- 플레이스 AI 브리핑 사례(리드젠랩 2025.12) — 체류시간 +10.4%, "더보기" 클릭률 +137%
- MDE Ranker (네이버 D2 알고리즘 원전) — 클릭 확률 + 만족 확률(체류시간) 동시 예측

### 현재 상태 (2026-05-26 기준)

- `src/contentGenerator.ts` L2448-2598 본문 합성 영역에 `subKeywords` 분배 + 분량 가이드가 일부 있음 (L2589-2596)
- 분야별 4구간 분리 가이드는 없음 (`baseLen` 하한 800자 정도만 명시)
- 카테고리 분류는 `homefeed/{health,it,...}.prompt` 13개 + `seo/{...}.prompt` 13개로 분리됨 — **보고서의 4분야 vs 코드의 13카테고리 매핑 필요**

## 3. 작업 분리 근거 (왜 별도 SPEC인가)

| 항목 | 제목 프롬프트 (R1~R3, 본 SPEC 외) | 본문 글자수 (본 SPEC) |
|---|---|---|
| 영향 영역 | `src/prompts/title/**`, `contentGenerator.ts` L640-690 (제목 합성) | `src/prompts/{seo,homefeed,affiliate}/**`, `contentGenerator.ts` L2448-2598 (본문 합성) + L820-848 (품질 평가) |
| 호출 흐름 | `generateTitleOnlyPatch` 단독 | `generateContent` → 본문 prompt 합성 → AI 응답 → 분량 검증 |
| 회귀 위험 | 텍스트 prompt 변경, 코드 영향 0 | **분량 검증 로직 추가 시 contentQualityChecker, AuthGR 등 다운스트림 영향** |
| 검증 방법 | vitest + 수동 1건 발행 | vitest + 분야별 발행 4건 + 체류시간 실측(네이버 서치어드바이저) |
| `no_cascade_fix` 카운트 | 1 fix (제목 영역) | 별도 1 fix (본문 영역) |

→ 단일 릴리즈에 묶으면 회귀 검증 게이트가 거대해지고 롤백 시 영향 영역이 모호해진다.

## 4. 범위

### In Scope
1. 분야 4분류 → 카테고리 13종 매핑 테이블 정의 (정보형/경험형/시즌형/펀팩트형)
2. `src/prompts/{seo,homefeed}/base.prompt`에 분야별 글자수 권장 블록 추가
3. `src/contentGenerator.ts` 본문 합성 로직에 분야 자동 판정 + 글자수 가이드 주입
4. `contentQualityChecker` 또는 별도 validator에 글자수 하한/상한 체크 추가 (-N점 패널티)
5. vitest 단위 테스트 (분야 판정, 글자수 검증)

### Out of Scope (별도 SPEC 또는 후속)
- 체류시간 실측 수집 (네이버 서치어드바이저 OpenAPI 연동 — SPEC-HOMEFEED-100 후속)
- 본문 구조 재설계 (소제목 5~7개, 단락 2~3문장 등은 별도)
- 이미지·캡션 분량 가이드 (`150~250자 캡션`은 별도)
- A/B 테스트 분야별 글자수 효과 측정 (SPEC-HOMEFEED-100 cohortAnalyzer 활용)

## 5. 제약

- **no_speculation**: 보고서 추측 효과("4배 노출", "체류 4분 도달") 인용 금지. 검증 가능 사실(MDE Ranker 평가 기준)만 인용.
- **no_cascade_fix**: 본 SPEC은 단일 릴리즈로 닫는다. 분야 4종 동시 적용. 회귀 검증 통과 못 하면 전체 롤백.
- **surgical change**: `contentGenerator.ts` L2448-2598 외 본문 로직 손대지 말 것.
- **언어 정책**: 프롬프트는 한글, 코드 주석은 영어.

## 7. 회귀 검증 게이트

1. vitest — 분야 판정 단위 테스트 + 글자수 검증 단위 테스트 (각 분야 1건씩 minimum)
2. 전체 vitest 통과 (flaky 무관 영역 제외)
