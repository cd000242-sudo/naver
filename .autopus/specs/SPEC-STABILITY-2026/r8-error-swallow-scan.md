## A급 우선순위 5대 수정 대상

### 1순위: A-1 (카테고리 선택 실패 → 무시)

**현상**: `selectCategoryInPublishModal()` 3회 재시도 모두 실패 시 로그만 출력 후 return

**코드**: publishHelpers.ts:440-447
```typescript
} catch (catError) {
  self.log(`   ⚠️ [시도 ${attempt}/${MAX_RETRIES}] 카테고리 선택 중 오류: ${(catError as Error).message}`);
  await closeCategoryDropdown();
  if (attempt === MAX_RETRIES) {
    self.log('   💡 기본 카테고리로 진행합니다.');  // ← 무시하고 진행!
  }
}
```

**위험**: 사용자가 "식품" 카테고리 요청 → 선택 실패 → 기본 카테고리로 자동 발행

**수정안**: `throw new Error('카테고리 선택 실패 — 발행 중단')`

### 2순위: A-2 (발행 모달 로드 타임아웃)

**현상**: 발행 모달 DOM이 5초 내 나타나지 않음 → "기본 카테고리로 진행" 로그, 카테고리 선택 단계 완전 스킵

**코드**: publishHelpers.ts:137-139

**위험**: 네이버 UI 렌더링 지연 시 사용자 지정 카테고리 무시

**수정안**: 타임아웃 후 throw 또는 재시도 loop (max 3회)

---

### 3순위: A-3 (임시저장 폴백 → 발행 → 임시저장 이중 모드)

**현상**: 발행 버튼 미발견 → 모달 닫기 + 임시저장 시도, 임시저장도 실패 → throw

**위험**: 발행 버튼 미발견(네이버 UI 변경) 시 사용자는 "발행 시도했는데 임시저장됐다" 혼동

**수정안**: 발행 버튼 미발견 시 즉시 throw (폴백 제거) 또는 사용자 컨펌 모달 표시

---

### 4순위: A-4 (URL 미변경 → 발행 완료 미확인)

**현상**: 발행 후 URL 미변경 + 성공 메시지 부재 → "추가 대기 후 재확인" → 5초 대기

**코드**: naverBlogAutomation.ts:1979-1988

**위험**: 발행 실패했는데 5초 대기 후 "발행 미확인" throw → 사용자가 재시도 → 이중 발행 가능

**수정안**: 성공 메시지 부재 시 throw (실패 취급), 재시도는 사용자 명시 요청 시만
