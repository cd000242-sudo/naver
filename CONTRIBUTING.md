# Contributing to Better Life Naver

Better Life Naver에 관심을 가져 주셔서 감사합니다. 작은 문서 수정부터 버그 재현, 테스트 추가, 안정성 개선까지 모두 환영합니다.

## 시작하기 전에

- 기존 Issue와 Pull Request에서 같은 문제가 논의 중인지 확인해 주세요.
- 큰 기능이나 동작 변경은 구현 전에 Issue에서 범위와 호환성을 먼저 논의해 주세요.
- 보안 취약점은 공개 Issue로 제보하지 말고 [SECURITY.md](SECURITY.md)를 따라 주세요.
- 실제 네이버 계정, API 키, 쿠키, 프록시 인증정보 및 사용자 콘텐츠를 커밋하지 마세요.

## 개발 환경

- Node.js 20.19 이상
- npm
- Windows 10/11, macOS 10.15 이상 또는 호환 Linux 개발 환경

```bash
npm install
npm run build
npm test
```

Electron 앱을 실행하려면 다음 명령을 사용합니다.

```bash
npm start
```

## 변경 작업 흐름

1. 하나의 Issue 또는 목적에 집중한 브랜치를 만듭니다.
2. 버그 수정과 기능 변경은 먼저 실패하는 테스트로 동작을 재현합니다.
3. 필요한 최소 구현을 추가하고 관련 테스트를 통과시킵니다.
4. 사용자 동작, 설정 또는 공개 API가 바뀌면 문서를 함께 수정합니다.
5. 제출 전 아래 검증 명령을 실행합니다.

```bash
npm run lint
npm test
npm run build
npm run verify:release
```

실제 발행 테스트에는 본인이 소유하거나 테스트 권한을 가진 계정만 사용하고, 기본적으로 임시저장 또는 비공개 테스트 환경을 우선해 주세요.

## 코드 원칙

- TypeScript 타입을 유지하고 입력 경계에서 값을 검증합니다.
- 기존 객체를 직접 변경하기보다 새 객체를 생성하는 방식을 선호합니다.
- 오류를 조용히 무시하지 말고 사용자에게 안전한 메시지와 진단 가능한 로그를 남깁니다.
- 함수와 파일의 책임을 작게 유지하고 기존 모듈 구조를 따릅니다.
- 비밀정보는 환경 변수나 안전한 저장소를 사용하며 로그에 원문을 남기지 않습니다.

## 커밋과 Pull Request

커밋 메시지는 Conventional Commits 형식을 사용합니다.

```text
fix: prevent duplicate publish requests
feat: add structured content quality gate
docs: clarify local development setup
test: cover session recovery timeout
```

Pull Request에는 다음 내용을 포함해 주세요.

- 해결하려는 문제와 변경 이유
- 주요 구현 내용과 영향 범위
- 실행한 테스트 및 결과
- UI 변경이 있다면 전후 스크린샷
- 보안, 개인정보, 계정 또는 외부 서비스에 미치는 영향
- 알려진 제한사항과 후속 작업

## Pull Request 체크리스트

- [ ] 변경 범위가 하나의 목적에 집중되어 있습니다.
- [ ] 관련 테스트를 추가하거나 기존 테스트가 충분한 이유를 설명했습니다.
- [ ] 린트, 테스트, 빌드 및 릴리스 검증을 실행했습니다.
- [ ] 실제 비밀정보와 개인 데이터가 포함되지 않았습니다.
- [ ] 필요한 README 또는 사용자 문서를 업데이트했습니다.
- [ ] 기존 동작과 하위 호환성에 미치는 영향을 설명했습니다.

검토 과정에서 수정 요청이 있을 수 있습니다. 합의된 범위 안에서 작고 검증 가능한 변경을 유지해 주세요.

## 기여 라이선스

Pull Request를 제출하면 별도의 서면 합의가 없는 한 해당 기여물을 이 프로젝트와 동일한 `AGPL-3.0-or-later` 조건으로 제공하는 데 동의하는 것으로 간주합니다. 제출자는 기여물을 해당 조건으로 제공할 권한이 있어야 하며, 라이선스가 불분명한 코드나 자산을 포함해서는 안 됩니다.

기여물의 저작권은 각 기여자에게 남습니다. 프로젝트 관리자가 기여물을 별도의 상용 라이선스로 제공하려면 해당 기여자와 별도의 서면 합의가 필요합니다. 현재 자동화된 CLA 절차는 마련되어 있지 않으므로, 별도 합의가 없는 외부 기여물은 상용 재라이선스 대상에 포함되지 않습니다.
