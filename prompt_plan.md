## 1. 2026 네이버 노출 기준 (리서치 종합 — 갭분석 기준선)

> 이 10개가 "최신 기준". 현재 프롬프트를 이것과 대조해 갭을 메운다.

1. **의미기반검색** (SMART BLOCK / Next N Search): 키워드매칭→의도매칭(임베딩). **키워드 반복은 오히려 -9%**. VIEW탭 폐지.
2. **C-Rank** (Creator Rank, 6요소): 블로그컬렉션 / 네이버DB정합 / SearchLOG(체류) / Chain(인용·공유) / Activity(발행주기) / 주제점수. + **AuthGR(권위)** 강화.
3. **D.I.A.+ v3** (HyperCLOVA X): **경험(Experience)이 핵심** — 직접경험·고유이미지 >> AI생성·정보나열. 이미지 원본성 정밀판별. 영상 가산점.
4. **체류시간**: 60초 최소 / **90초+ & 스크롤 80%+ → 상위 2.4배** / 30초 이하 이탈 → 하락.
5. **최신성·활동성**: 최근 30일 활동 부족 → 노출 급감. **주 2~3회 정기 간격** 발행 > 몰아쓰기.
6. **AEO / AI브리핑 / Cue:** (제로클릭·Position Zero): 첫문단 + **각 H2 첫 100자 내 직답(40~80자)** / **질문형 H2** / **FAQ 5~8개** / 구체수치·통계·출처(전문가인용 **+41%**, 통계 **+30%**, 인라인출처 **+30%**) / 비교·vs·TopN **2.3배**. Cue:는 VIEW·지식인·플레이스 소스.
7. **홈판(홈피드)**: **0.8~1.2초 안에 스크롤 멈춤**이 전부. **첫 3줄 + 대표이미지** 훅이 성패. 초기 1~3초 상호작용 1차필터. 발행후 24~48h 좋아요/댓글/공유가 확산결정. **호기심형 제목**(검색용과 별개). 대표이미지 1080×1080+, 텍스트 1/4 이하. **투트랙**(검색형+홈피드형).
8. **제목**: 메인키워드 맨앞 / 숫자형 **CTR +36%** / 질문·비교·방법·최신형 / 구체숫자+결과약속+대상명시 중 2개+ / 네이버 30~40자.
9. **수익**: 주제 1~3개 집중(C-Rank·광고단가) / 2,000~3,000자 / 광고위치(도입+중간+Q&A) **1.5~2배** / 카테고리 정확분류→단가↑ / 롱테일 전환↑ / CPA(쿠팡파트너스) 병행.
10. **저품질 회피**: 동일이미지 3회+, 외부링크 5개+, 해시태그 10개+, 키워드 15회+, 하루 10개+ 발행.

## 1.5 노출·인용 극대화 구조 사양 (전 엔진 강제 + self-critique 검증)

> **왜 구조가 핵심**: Cue:/AI브리핑은 글을 passage(문단·문장) 단위로 추출·인용 → **각 chunk가 완결된 답이면 그것만 인용**. 홈판은 첫 chunk+대표이미지로 0.8초 판정. 구조 미흡 = 내용 좋아도 인용/노출 실패.

| 요소 | 사양 | 겨냥 |
|---|---|---|
| A. 제목 | 키워드 앞 + L2 상황 함축 + 호기심격차, 30~40자 | 검색+홈판 |
| B. 대표이미지 | **1:1 정방형 800px+**(16:9 금지=목록 잘림), 텍스트 1/4 이하, 고대비, 첫 소제목 정렬 (상세 §1.6) | 홈판 0.8초 |
| C. 도입부 | [상황 L2 → 공감 → 현상+직답(첫100자) → 가치+궁금증] 첫3줄=훅 | 홈판+체류+AEO |
| **D. 본문=청크 Q&A** | ①H2 질문형/명확키워드 ②**H2 첫100자 직답(40~80자)=완결 chunk** ③구체수치·날짜+인라인출처([자료N]) ④비교/선택→표(2열)·TopN(2.3배) ⑤소제목당 시각요소 1개 | **인용 핵심** |
| E. FAQ | 5~8개 자연어 Q&A | 인용 |
| F. 한 문장 요약 | 핵심 55~80자 블록 | AI 발췌 |
| G. 마무리 | 내부링크(Chain) + 댓글/공유 유도 CTA | 홈판 초기반응+체류 |
| H. 저품질 회피 | 키워드 <15, 외부링크 <5, 해시태그 8~12, 동일이미지 금지 | 패널티 회피 |

> **네이버 특수성**: FAQPage JSON-LD 스키마는 웹용 — 네이버 스마트에디터는 스키마 불가. → 텍스트 Q&A 구조(질문형 소제목+직답)로 대체. 그래서 D·E·F 본문 구조가 더 중요.

## 1.6 이미지 노출·인용 사양 (정직한 양날의 검 — AI 이미지)

> **전제: API/AI 이미지 생성이 핵심 도구. "AI 쓰지 마라"가 아님.** D.I.A.+가 실제로 저품질 처리하는 트리거는 "AI인지"보다 **중복·동일패턴**(동일 이미지 재사용 / 미리캔버스류 동일포맷 / 같은 로고·텍스트 반복 — 2025 이미지 유사문서 판독 강화). → **글마다 고유한 AI 이미지를 새로 생성하면 위험 대부분 해소**(앱 기존 동작이 유리). 핵심은 회피가 아니라 **고유성·다양성·규격·ALT 최적화**(전부 API/자동). 실사·캡처는 강제 아닌 선택 부스트. (장기: C2PA·SynthID 워터마크 확산은 모니터링 대상)

| 요소 | 사양 |
|---|---|
| 썸네일 비율 | **1:1 정방형 800×800px+** (16:9 금지 — 목록 위아래 잘림) |
| 파일 | PNG, 2배 해상도, 1MB 이하, 모바일 120px서도 선명 |
| 톤 | 밝고 선명·고대비, 톤/색감 통일 |
| **고유성** | 글마다 새 이미지, 동일 패턴/포맷/로고·텍스트 반복 금지 |
| **ALT** | 키워드 포함 ALT 자동 생성 → D.I.A.+ 가산점 + 이미지-텍스트 일관성 |
| 영상/GIF | 본문 클립/움짤 1개 → 가산점(AI 약점 보완) |
| 실사 옵션 | 직접촬영 5장+은 추가 부스트(강제 아님) — 원하는 사용자만 |

**⭐ 모드별 이미지 "타입" 분기 (전부 API/codex 자동 생성)**:
| 글 유형 | 생성할 이미지 타입 | 경로 |
|---|---|---|
| 경험형(맛집/여행/리뷰/일상) | 장면 이미지(AI) — 고유·다양 | API 이미지 생성 |
| **정보성(정책/세금/금융/방법/비교)** | **인포그래픽·비교표·요약 도식 스타일** (AI 풍경 사진 ❌, 정보가치 0) | API 인포그래픽 생성 또는 codex HTML 표(Phase 3) |
| 상품/쇼핑 | 상품 이미지 + 비교표 | API + codex 표 |

> **핵심 전환**: 조정은 "AI를 쓰냐 마냐"가 아니라 **"어떤 타입을 생성하냐"**. 정보성 글엔 AI 사진 대신 **API 인포그래픽 / codex 표**를 생성(고유·자동·무료·D.I.A.+/AEO 강). 수동 캡처(공식문서·신청화면)는 원하는 사용자만 슬롯에 추가하는 **선택 부스트**.

**AI 이미지 저품질 회피 (전부 API/자동)**:
1. 고유성 보장(글마다 새 생성 — 앱 기존 동작) 2. "원본스러움" 후처리(약한 노이즈/크롭/색보정, sharp) 3. ALT 자동화(imageTextConsistencyChecker) 4. 모드별 타입 분기(정보성=인포그래픽/표) 5. 규격 강제(1:1 800px+ PNG).

> **정직한 참고**: D.I.A.+ 경험·원본 가산은 실사가 최강이라, 실사를 *원하는* 사용자에겐 부스트가 됨(강제 아님). C2PA/SynthID는 픽셀 워터마크라 EXIF 제거로 안 지워질 수 있어 장기 모니터링. → 기본 전략 = **API 이미지 고유성·타입·ALT·규격 극대화로 충분**, 실사는 선택.

**🔴 심각한 갭 (Phase 2 우선)**
- G1. **에이전트 모드에 mode-프롬프트 전달 검증 필요** — codex/claudeRunner가 네이버 모드기준을 확실히 받는지 (인벤토리상 의심구간). 미전달 시 "기본 블로거 페르소나"로 작동 → 노출신호 미최적화.
- G2. **AEO 첫 100자 직답이 "각 H2마다" 강제 안 됨** — 도입 두괄식은 있으나 H2별 청크 직답 규칙 부재(인용 1순위 조건).
- G3. **FAQ 섹션이 SEO/홈판 base에 표준화 안 됨** — 메이트만 FAQ 강제. AEO상 전 정보성 글에 FAQ 5~8개 필요.
- G4. **출처 신뢰도 우선순위 미정의** — `[자료N]` 토큰은 있으나 뉴스>공식>블로그>지식인 우선순위·분산위치 규칙 없음.

- G9. **"깊은 상황-공감" 구조 부재** (사용자 실측 통찰, 2026 기준 정합) — 상황은 제목 패턴이 아니라 **본문 공감 구조**: 글에 [검색자 상황 명시 + 문제 공감 + 현상(왜 그런지) 설명 → 해결]이 함께 전개돼야 함.
  - **깊이 정의(핵심)**: 표면적 상황(L1, 누구나 예상)이 아니라 **L2 = "누구나 겪지만 아무도 안 짚은 숨은 층위"** = 보편성(검색수요) × 깊이(강한 공감)의 교집합. 흔한 경험의 안 짚힌 트리거.
  - **제목**: 깊은 상황을 **함축적·간결하게** 신호(키워드는 명시, 상황은 암시, 호기심 격차, 네이버 30~40자 — 다 설명해 길어지면 실패).
  - 현재 도입부(GAMMA-7 4축)는 호기심엔 강하나 "검색자 상황+공감+현상설명 3종 동시" + "L2 깊이"를 명시 강제 안 함. 단일 고의도 롱테일 → 의미기반검색·체류·AEO·D.I.A.+ 경험신호 동시 충족. 정적 템플릿 불가, 에이전트 추론 적격.

**🟡 중간 갭 (Phase 2 후순위)**
- G5. 홈판 "0.8초 훅" 정량 목표 부재(첫줄 25자만 규정). 대표이미지 규격(1080²·텍스트1/4) 프롬프트 미반영.
- G6. 체류 90초/스크롤80% 목표를 구조(표·이미지 리듬·소제목 간격)로 역설계한 규칙 없음.
- G7. 수익: 카테고리별 광고위치/고단가 키워드 매칭이 SEO에만, 홈판/메이트 미적용.
- G8. 이미지-본문 일관성 자동감지 루프 미가동(imageTextConsistencyChecker).

## 3. Phase 설계

## A. DropShot 로그인 수명주기

1. 로그인 감지에 성공하면 보이는 로그인용 브라우저 컨텍스트가 완전히 닫힐 때까지 기다린 뒤 성공을 반환한다.
2. 로그인 확인 직후에는 새 브라우저를 다시 열지 않는다. 실제 이미지 생성 시점에만 백그라운드 컨텍스트를 단일 생성한다.
3. 생성 도중 대화형 로그인이 필요했던 경우에는 보이는 컨텍스트 종료 완료 후 백그라운드 컨텍스트로 한 번만 인계한다.
4. 기존 DropShot 페이지가 있으면 재사용하고, 함께 생성된 중복 `about:blank` 페이지만 정리한다.
5. 상태 확인·로그인·생성 작업의 기존 중복 실행 방지 장치를 유지한다.

## B. 공정위 문구 원문 보존

1. 쇼핑커넥트 모드 진입은 공정위 문구를 켜기만 하며, 저장된 프리셋·직접 입력 문구를 기본 프리셋으로 덮어쓰지 않는다.
2. 사용자가 프리셋을 직접 바꾸거나 기본값 복원 버튼을 누른 경우에만 문구를 변경한다.
3. AI 프롬프트에는 공정위/광고 고지를 생성·요약·변형하지 말고 앱이 확정 문구를 삽입한다는 규칙을 명시한다.
4. 발행 인계 과정에서는 사용자가 확정한 문구를 그대로 유지하며 품질 보정·정규화 대상에서 제외한다.

## C. 회귀 게이트

1. 로그인 성공 후 보이는 컨텍스트 종료, 중복 빈 탭 정리, 생성용 백그라운드 인계를 단위 테스트로 고정한다.
2. 모드 왕복 후 사용자 문구 보존, 명시적 프리셋 변경만 허용, AI 임의 고지 금지를 테스트로 고정한다.

4. 모든 게이트를 통과한 버전만 버전 상승·커밋·푸시·GitHub 릴리스한다.
