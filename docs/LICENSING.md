# Licensing Guide

Better Life Naver uses an open-source license for the project's original code while preserving the option to discuss separate commercial terms for code whose copyright is controlled by the project maintainer.

## Open-source license

Unless a file or directory states otherwise, original Better Life Naver code is licensed under the [GNU Affero General Public License, version 3 or any later version](../LICENSE) (`AGPL-3.0-or-later`).

The AGPL permits commercial and non-commercial use, copying, modification, and redistribution under its conditions. Among other obligations, distributors must provide the corresponding source, and operators of a modified version that supports remote network interaction must offer that version's corresponding source to its remote users. Read the full license before relying on this summary; the license text controls if this guide differs from it.

## Alternative commercial terms

Alternative commercial terms may be available for portions of the project for which the maintainer holds the necessary rights. Such terms can be useful when an organization cannot comply with the AGPL's conditions. See [COMMERCIAL-LICENSING.md](../COMMERCIAL-LICENSING.md) for the limits of this option.

An alternative commercial license does not change or replace the licenses of FFmpeg, npm packages, copied assets, external contributions, or any other third-party material.

## Contributions

Contributions accepted without a separate written agreement remain owned by their contributors and are provided under `AGPL-3.0-or-later`. The project does not assume a right to relicense a contributor's work commercially merely because a Pull Request was accepted.

## Third-party material

Third-party components remain governed by their respective licenses. In particular, several directories under `skills/` contain their own `LICENSE.txt`; those notices control the files in their respective directories. See [THIRD_PARTY_NOTICES.md](../THIRD_PARTY_NOTICES.md), dependency metadata, and any license files located next to individual components.

The distributed `ffmpeg-static` package includes an FFmpeg executable built with GPLv3-enabled components. Anyone distributing application binaries must satisfy the applicable GPL source, notice, and license obligations in addition to the AGPL obligations for project code.

## Product access and trademarks

The open-source license covers copyright permissions in the code. It does not promise access to hosted services, API credentials, official update infrastructure, support, or paid product features. Names, logos, and other branding are addressed separately in [TRADEMARKS.md](../TRADEMARKS.md).

This guide is practical project documentation, not legal advice.
