# UX/UI 감사 — Better Life Naver (2026-06)

> 목적: 코딩 전에 현 UI/UX를 비평하고 "모던·고급·초보자 친화" 리뉴얼 방향을 우선순위로 합의.
> 원칙: 빅뱅 재작성 ❌(13,000줄 god-file 회귀 위험) → 단계적 리프레시 ✅.

## 0. 총평 (Conclusion First)

**기능은 매우 강력하지만 UI는 "유기적으로 비대해진 전문가 도구"** — 정보 밀도가 높고, 피드백이 종종 불안을 유발하며, 초보자 동선이 없다. 리뉴얼의 핵심은 **새 기능 추가가 아니라 "덜어내기 + 동선 정리 + 일관성"**. 현 사용자(부업러 다수·초보 포함)에겐 "강력함"보다 "안 무섭고 명확함"이 전환을 만든다.

## 1. 현재 구조 진단

- **최상위 탭(5)**: 🏠 메인 / 🚀 스마트 자동 / 🎨 썸네일·배너 / 🖼️ 이미지 / 📊 분석
- **헤더 버튼(3)**: 전체 초기화 · 트레이 숨기기 · 브라우저 종료
- **모달 다수**: 비용표·AI설정·연속발행·다중계정 풀오토 등 핵심 작업이 모달에 흩어짐
- **스타일**: 인라인 스타일 위주, 디자인 토큰(`--text-gold` 등) 일부만 사용 → 화면마다 간격·색·버튼 크기 제각각

## 2. 심각도별 발견

### 🔴 Critical (전환·이탈 직결)
1. **정보 밀도 과부하** — 비용표 모달은 5단계 표·카드가 한 화면에 쏟아짐. 초보자는 "복잡하다=어렵다=포기". → 기본은 추천 1줄, 상세는 "자세히"로 접기.
2. **불안 유발 피드백** — "콘텐츠 생성 지연" 주황 ⚠️ 배너가 정상 대기 중에도 에러처럼 보임(이미 제거 커밋). 진행 표시는 "작동 중"을 안심시켜야지 경고하면 안 됨.
3. **초보자 온보딩 부재** — 첫 실행 시 "무엇부터?"가 없음. API 키·엔진·발행 설정을 스스로 찾아야 함 → 첫 글까지 도달률 낮음.

### 🟠 High (사용성)
4. **모달 남용** — 연속/다중계정 발행이 모달 → 컨텍스트 단절·산만. (사용자 요청대로) 탭/서브탭으로 인라인화가 맞음.
5. **탭 명명 불명확** — "메인"이 무엇인지 모호. 기능 기반 명명("글 발행하기" 등)이 직관적.
6. **버튼 위계·크기 불일치** — 같은 화면에 큰 버튼·작은 텍스트 혼재. 주요 동작은 크고 명확하게, 보조는 작게.
7. **헤더 버튼 혼란** — "트레이 숨기기"는 창 내리기와 중복, "브라우저 종료"는 일반 사용자에게 의미 불명.

### 🟡 Medium (완성도)
8. **숨은 중요 정보** — 비용표·추천이 묻혀 안 보임(톱니/재오픈 버튼으로 개선 중).
9. **시각 일관성** — 색·radius·그림자·여백이 화면마다 달라 "고급감" 저하.
10. **빈 상태(empty state) 안내 부족** — 생성 글 목록·이미지 목록이 비었을 때 다음 행동 유도 없음.

## 3. 사용자 요청 ↔ UX 원칙 매핑

| 사용자 요청 | UX 원칙 | 평가 |
|---|---|---|
| 탭 재구성(기능 기반 명명) | 인식 용이성(Recognition) | ✅ 옳음 |
| 연속/다중계정 모달→서브탭 | 컨텍스트 유지 | ✅ 옳음 |
| 버튼 크게 + rename | 위계·가독성 | ✅ 옳음 |
| 공지 센터 + 관리자 편집 | 운영자 소통·신뢰 | ✅ 유용(단 보안 주의) |
| 트레이 버튼 제거 | 중복 제거(Hick's Law) | ✅ 옳음 |
| 브라우저종료→사용법 영상 | 초보자 학습 동선 | ✅ 옳음 |
| 구매→1:1 카톡 | 마찰 낮은 문의 | ✅ 옳음 |

→ 사용자 직관이 UX 원칙과 대부분 일치. 신뢰하고 진행 가능.

## 4. 우선순위 로드맵 (단계적, 각 게이트·커밋)

### Phase 1 — 구조(탭) 리프레시 [높은 가치·높은 위험 → 신중]
- 최상위 탭: **시간/달력·메모장 | 글 발행하기 | 이미지 생성/관리 | 썸네일 배너 생성기** + **생성된 글 목록**(신규, 기존 목록 이동)
- "글 발행하기" 안에 **서브탭**: 단일발행 / 연속발행 / 다중계정 풀오토 — 연속·다중계정 모달을 서브탭에 인라인 임베드
- 분석/가이드는 "가이드·분석 모음"으로 정리

### Phase 2 — 헤더·버튼 정리 [낮은 위험]
- 헤더 버튼 6종 확대+rename: 전체 초기화 / 환경설정 / 가이드·분석 모음 / 1:1 카톡문의 / 앱 사용법 영상 / 황금키워드 찾기
- 트레이 숨기기 제거(창 내리기로 대체) · 브라우저 종료→유튜브 사용법 링크 · 구매문의→1:1 카톡문의
- Flow 불가 안내문구(제미나이 Plus 확인 + 딥인프라/프로디아 + 레오나르도 1:1)

### Phase 3 — 공지+관리자 [신규 기능]
- 로그인 후 센터 공지 배너 + 톱니바퀴 관리자모드(소프트 게이트) → 공지 편집(글크기/꾸미기/이미지)+FAQ
- ⚠️ 비번 클라이언트 하드코딩은 "소프트 게이트"(진짜 보안 X) — 운영 편의용으로만

### Phase 4 — 에이전트 계정 전환 [낮은 위험]
- 로그아웃/전환 버튼(codex/claude logout→재로그인)

### Phase 5 — 디자인 시스템 폴리시 [점진]
- 디자인 토큰 통일(색·radius·간격·버튼 크기 스케일) → 화면마다 제각각 해소
- 빈 상태·온보딩(첫 글까지 3스텝 가이드) → 초보자 도달률↑

## 5. 모던·초보자 친화 방향 (제안)

1. **"기본은 단순, 상세는 접기"** — 모든 복잡 패널은 추천 1줄 + "자세히". 초보자는 추천만 보고 시작.
2. **명확한 1차 동작(Primary CTA)** — 화면당 "지금 할 일" 버튼 1개를 가장 크고 눈에 띄게.
3. **안심형 피드백** — 진행은 "작동 중"으로, 경고색(주황·빨강)은 진짜 문제일 때만.
4. **일관 디자인 토큰** — 색·여백·버튼 크기·radius를 토큰으로 통일 → 즉시 "고급감".
5. **온보딩 3스텝** — (1) 엔진/키 (2) 키워드 (3) 발행. 첫 글 도달을 가이드.
